## Schema-Instruction Mismatches

### Issue 1: Resolution enum mismatch for CDP/ATP terminal paths

**Affects:** EXEC-05, OUTP-02
**Location:** SKILL.md line 199

SKILL.md instructs `reframe_and_close` as the terminal resolution for all skip-retry diagnoses. However:
- **CDP** (`construct_not_decomposable`): Schema defines `close_as_unified`, not `reframe_and_close`
- **ATP** (`transfer_not_viable`): Schema defines `close_as_rejected`, not `reframe_and_close`
- **CFFP** (`construct_incoherent`): Correctly has `reframe_and_close`

**Impact:** Narrative output unaffected. `--structured` output would produce invalid resolution enum values for CDP/ATP terminal paths.

### Issue 2: ADP missing #Protocol.version field

**Affects:** OUTP-03
**Location:** SKILL.md line 308, adp.opt.cue

SKILL.md instructs reading `#Protocol.version` for the `source_run.run_version` field in `#Record` output. ADP has no `#Protocol` type and no `version` field — all other 12 protocols do.

**Impact:** ADP + `--record` flow cannot populate `source_run.run_version` from schema.

### Issue 3: AAP FragilityMap tier label mismatch

**Affects:** EXEC-02, OUTP-02
**Location:** SKILL.md line 231, aap.opt.cue line 163

SKILL.md describes tiers as: `load-bearing`, `structural`, `background`
Schema defines: `structural`, `significant`, `moderate`, `minor`

**Impact:** Both narrative and `--structured` AAP output use incorrect tier labels.

### Issue 4: #ProtocolInstance type does not exist (tech debt)

**Affects:** OUTP-02
**Location:** SKILL.md line 283

SKILL.md references `#ProtocolInstance` but actual types are `#{ACRONYM}Instance` (e.g., `#CFFPInstance`). Not a runtime break — Claude infers the correct type — but the instruction is imprecise.
