**File 1: `socrates/hooks/hooks.json`**
Plugin hooks configuration. Must use the `"hooks"` wrapper format (not the settings.json direct-key format). Content:

```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "startup|resume|clear",
        "hooks": [
          {
            "type": "command",
            "command": "\"${CLAUDE_PLUGIN_ROOT}/hooks/session-start\""
          }
        ]
      }
    ]
  }
}
```

Key points:
- Matcher is `startup|resume|clear` per locked decision (no `compact`)
- Command path uses `${CLAUDE_PLUGIN_ROOT}` which Claude Code expands in the command string (this works correctly per research)
- Quotes around the command handle paths with spaces
- No `run-hook.cmd` wrapper — superpowers deprecated it; extensionless script called directly
- No `description` field needed at top level (optional)

**File 2: `socrates/hooks/session-start`**
Extensionless bash script (NO `.sh` extension — required to avoid Claude Code 2.1.x auto-prepend behavior on Windows). Must be chmod +x.

The script must:
1. Use `#!/usr/bin/env bash` shebang
2. Derive plugin root via `BASH_SOURCE[0]` (NOT `$CLAUDE_PLUGIN_ROOT` which is unset in hook shell — bug #24529)
3. Locate SKILL.md at `${PLUGIN_ROOT}/skills/socrates/SKILL.md`
4. Silent failure: if SKILL.md missing or no frontmatter found, `exit 0` with no output
5. Extract YAML frontmatter using `awk '/^---$/{c++;next} c==1{print}'` (POSIX, macOS-safe — do NOT use sed which has BSD/GNU differences)
6. JSON-escape the frontmatter using bash parameter substitution (backslashes, double quotes, newlines, carriage returns, tabs)
7. Output `hookSpecificOutput` JSON via `printf` (NOT `echo` — echo behavior varies by shell/platform)
8. Always `exit 0`

Reference implementation from RESEARCH.md Pattern sections — follow it closely:

```bash
#!/usr/bin/env bash
# hooks/session-start — extensionless intentionally (Claude Code 2.1.x .sh auto-prepend avoidance)
# Injects SKILL.md YAML frontmatter as additionalContext for SessionStart hook

# Derive plugin root from script location (CLAUDE_PLUGIN_ROOT is unset in hook env, bug #24529)
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]:-$0}")" && pwd)"
PLUGIN_ROOT="$(cd "${SCRIPT_DIR}/.." && pwd)"
SKILL_PATH="${PLUGIN_ROOT}/skills/socrates/SKILL.md"

# Silent failure: file missing
if [ ! -f "$SKILL_PATH" ]; then
  exit 0
fi

# Extract YAML frontmatter (content between first --- pair)
frontmatter=$(awk '/^---$/{c++;next} c==1{print}' "$SKILL_PATH")

# Silent failure: no frontmatter found
if [ -z "$frontmatter" ]; then
  exit 0
fi

# JSON-escape the frontmatter (bash parameter substitution — no external tools)
escaped="${frontmatter//\\/\\\\}"    # backslashes first
escaped="${escaped//\"/\\\"}"         # double quotes
escaped="${escaped//$'\n'/\\n}"       # newlines
escaped="${escaped//$'\r'/}"          # strip carriage returns
escaped="${escaped//$'\t'/\\t}"       # tabs

# Output hookSpecificOutput JSON
printf '{"hookSpecificOutput":{"hookEventName":"SessionStart","additionalContext":"%s"}}' "$escaped"
exit 0
```

Do NOT use `set -euo pipefail` — it conflicts with the silent failure design (set -e would exit 1 on any failure, showing as error in verbose mode).

After creating the file, run: `chmod +x socrates/hooks/session-start`

**File 3: `.gitattributes`** (at repo root, not inside socrates/)
Enforce LF line endings for hook scripts to prevent Windows CRLF corruption:

```
# Normalize line endings
* text=auto

# Force LF for shell scripts and hook files

socrates/hooks/* text eol=lf
```

Note: Use `socrates/hooks/*` (not just `hooks/*`) since the repo root is one level above the plugin root. The `*.sh` rule is belt-and-suspenders for any future .sh files. The `* text=auto` normalizes all text files.
