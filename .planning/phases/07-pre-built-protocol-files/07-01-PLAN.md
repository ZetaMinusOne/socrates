  <action>
    Update the existing Makefile to:
    1. Add `.PHONY: build clean check` declaration (currently only has `.PHONY: build clean`)
    2. Add a `check` target that regenerates protocol files in-place then uses `git diff --exit-code` to detect staleness

    The updated Makefile should be:

    ```makefile
    .PHONY: build clean check

    build:
    	python3 scripts/strip_cue.py

    clean:
    	find socrates/protocols -name '*.opt.cue' -delete

    check:
    	python3 scripts/strip_cue.py
    	@git diff --exit-code socrates/protocols/ > /dev/null 2>&1 && echo "Protocol files are up to date." || (echo "Protocol files are stale — run 'make build' and commit the changes."; exit 1)
    ```

    The check target works by: (1) regenerating all files from current submodule state (idempotent), (2) checking if any tracked files differ from the regenerated versions. If the submodule hasn't changed, `strip_cue.py` produces identical output and `git diff` shows nothing. If the submodule was updated, differences appear and the developer is prompted to commit.

    Stage both files: `git add Makefile scripts/strip_cue.py`

    Commit with message: `feat(07): add make check staleness target and track build infrastructure`

    Do NOT modify `scripts/strip_cue.py` — it is already fully functional. Just track it in git.
  </action>
